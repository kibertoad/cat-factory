Document the `@<key>` translator-description convention in CLAUDE.md's i18n section:
when (and mostly when not) to annotate an `en` catalog key. Emphasizes that the default
is no description — annotate only genuinely ambiguous keys (homographs, do-not-translate
proper nouns, umbrella titles, placeholder/format constraints, extra plural forms) so the
few notes that matter aren't drowned in noise. Docs-only.
